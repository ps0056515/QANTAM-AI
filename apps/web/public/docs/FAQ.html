General Questions

What is QANTAM AI?

QANTAM is an AI-powered quality engineering platform that helps QA teams work faster and smarter.

It consists of 5 integrated modules:

  1. CLARITY - Analyzes requirements for ambiguity, gaps, and conflicts
  2. PULSE - Generates test cases automatically
  3. SIGNAL - Calculates release quality scores for Go/No-Go decisions
  4. FORGE - Flags risky code changes in pull requests
  5. SENTINEL - Traces production bugs back to test gaps

All modules share a central intelligence graph (QKG) that gets smarter over time.

What makes QANTAM different from other QA tools?

Three key differentiators:

  1. Quality Knowledge Graph (QKG) - All modules share a central intelligence graph that connects requirements → tests → code → defects → production incidents.
  2. Confidence Scoring - Every AI output includes a confidence score (0-100). Low-confidence items are flagged for human review.
  3. Accuracy Gates - No AI feature ships until it passes precision/recall thresholds on golden datasets.
The Compounding Effect: When SENTINEL traces a production bug to a test gap, that feedback improves PULSE's test generation. This is the flywheel competitors don't have.

  • Release documentation: 90% faster (5 min vs 2 hours)

Architecture & Technical

What is the Quality Knowledge Graph (QKG)?

The QKG is a Neo4j graph database that serves as the "brain" of QANTAM. It stores relationships between:

  • Requirements - With quality scores, flags, and regulatory mappings
  • Test Cases - Linked to source requirements via traceability seeds
  • Code Modules - With defect history and risk scores
  • Defects - Categorized by severity and root cause
  • Production Incidents - Traced back to test and requirement gaps
(:Requirement)-[:REQUIRES_TESTING]->(:TestCase) (:TestCase)-[:COVERS]->(:CodeModule) (:CodeModule)-[:HAS_DEFECT]->(:Defect) (:Defect)-[:ESCAPED_AS]->(:ProductionIncident)

Modules

What does CLARITY do?

CLARITY is the Requirement Intelligence module.

What it detects:

  • Vague Language - "The system should be fast" → What is "fast"?
  • Undefined Terms - "User" without specifying which type
  • Missing Actors - "Data should be validated" → By whom?
  • Multiple Interpretations - Ambiguous statements
  • Untestable Requirements - No clear pass/fail criteria
  • Conflicts - Requirements that contradict each other

Output: Requirement Quality Score (RQS) 0-100, flags with suggested rewrites, Health Report PDF

What does PULSE do?

PULSE is the Test Suite Factory.

Test types generated: Positive, Negative, Edge Cases, Compliance Tests

Export formats: Jira Zephyr Scale, Azure DevOps, Gherkin, Excel, CSV

Human review: Every test goes through Approve / Edit / Reject. Feedback improves future generation.

What does SIGNAL do?

SIGNAL calculates Release Quality Score (RQS) for Go/No-Go decisions.

RQS Formula:

  • Coverage Score (35%) - % requirements with approved tests
  • Defect Score (30%) - Open P0/P1/P2 count
  • Test Execution Score (25%) - Pass rate
  • CLARITY Score (10%) - Average requirement quality
ScoreDecision
> 80GO
60-80CONDITIONAL GO
< 60NO-GO
What does FORGE do?

FORGE provides Code Review Intelligence.

It analyzes PRs and posts Quality Risk Score comments within 60 seconds based on:

  • Module defect density (last 90 days)
  • Change complexity
  • Coverage delta
  • Days since last test update
What does SENTINEL do?

SENTINEL monitors production and traces bugs back to test gaps.

  • Log Ingestion from APM tools
  • Escaped Defect Classification
  • QKG Tracing: incident → test gap → requirement gap
  • Auto Regression Test generation
The Compounding Loop: SENTINEL findings feed back into QKG, making FORGE and PULSE smarter over time.

Integration & Legacy Systems

How does QANTAM work with legacy projects?

QANTAM integrates incrementally without requiring a complete overhaul:

Phase 1: Connect (Week 1) - Connect Jira/ADO, GitHub/GitLab, upload existing docs

Phase 2: Ingest (Week 2-3) - Import requirements, test cases, defect history into QKG

Phase 3: Use (Week 4+) - CLARITY analyzes new requirements, PULSE generates tests, FORGE comments on PRs

Key Point: You don't need to re-test everything. QANTAM works incrementally on new work while learning from historical data.

Security & Multi-Tenancy

How does multi-tenant isolation work?

Strict isolation at every layer:

  1. JWT Token - Every request includes tenant_id
  2. Middleware - Validates tenant_id on every request
  3. Database - Every query includes tenant_id filter
  4. Neo4j - QKG queries always scoped by tenant_id

Cross-tenant access is architecturally impossible.

Is data sent to external AI providers?
  • Claude (Anthropic) - Used for complex analysis. Data sent to Anthropic API.
  • Llama 3.1 (local) - Used for simpler tasks. Data stays on your infrastructure.

For strict data residency: on-premise deployment, all-local-LLM mode, or private Claude via AWS Bedrock available.
